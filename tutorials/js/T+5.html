Introduction to Operators & Expressions

In JavaScript, operators are special symbols that perform operations on operands (values or variables). An expression is a combination of values, variables, operators, and function calls that evaluates to a single value.

// Simple expression with arithmetic operator
let sum = 5 + 3; // '+' is the operator, 5 and 3 are operands
// The entire "5 + 3" is an expression that evaluates to 8

JavaScript supports various types of operators that allow you to perform arithmetic, assign values, compare values, combine logical expressions, and more.

Arithmetic Operators

Arithmetic operators perform mathematical operations on numeric values (literals or variables).

Operator Name Example Result
+ Addition 10 + 5 15
- Subtraction 10 - 5 5
* Multiplication 10 * 5 50
/ Division 10 / 5 2
% Modulus (Remainder) 10 % 3 1
** Exponentiation 2 ** 3 8
++ Increment x++ x + 1
-- Decrement x-- x - 1
// Arithmetic operator examples
let a = 15, b = 4;

let addition = a + b; // 19
let subtraction = a - b; // 11
let multiplication = a * b; // 60
let division = a / b; // 3.75
let remainder = a % b; // 3 (15 divided by 4 is 3 remainder 3)
let exponent = a ** 2; // 225 (15 to the power of 2)

// Increment and decrement
a++; // a is now 16 (post-increment)
++b; // b is now 5 (pre-increment)

Important Note

The increment (++) and decrement (--) operators can be used in both prefix (++x) and postfix (x++) positions. The prefix form returns the value after incrementing, while the postfix form returns the value before incrementing.

Assignment Operators

Assignment operators assign values to JavaScript variables. The basic assignment operator is =, but there are compound assignment operators that combine arithmetic with assignment.

Assignment =

Assigns a value to a variable

Add & Assign +=

Adds then assigns: x += y is x = x + y

Subtract & Assign -=

Subtracts then assigns: x -= y is x = x - y

Multiply & Assign *=

Multiplies then assigns: x *= y is x = x * y

Divide & Assign /=

Divides then assigns: x /= y is x = x / y

Modulus & Assign %=

Remainder then assigns: x %= y is x = x % y

Exponent & Assign **=

Exponentiates then assigns: x **= y is x = x ** y

// Assignment operator examples
let x = 10; // Basic assignment

// Compound assignment operators
x += 5; // x is now 15 (same as x = x + 5)
x -= 3; // x is now 12 (same as x = x - 3)
x *= 2; // x is now 24 (same as x = x * 2)
x /= 4; // x is now 6 (same as x = x / 4)
x %= 5; // x is now 1 (same as x = x % 5)
x **= 3; // x is now 1 (same as x = x ** 3)

Pro Tip

Compound assignment operators not only make your code more concise but can also improve performance in some cases, as the variable is evaluated only once.

Comparison Operators

Comparison operators compare two values and return a boolean value (true or false). They are used in conditional statements and loops.

Operator Description Example Returns
== Equal to (loose equality) 5 == "5" true
=== Strict equal to (value and type) 5 === "5" false
!= Not equal to (loose inequality) 5 != "6" true
!== Strict not equal to 5 !== "5" true
> Greater than 10 > 5 true
< Less than 10 < 5 false
>= Greater than or equal to 10 >= 10 true
<= Less than or equal to 10 <= 5 false
// Comparison operator examples
let num1 = 10, num2 = 5, strNum = "10";

let isEqualLoose = num1 == strNum; // true (loose equality, type coercion)
let isEqualStrict = num1 === strNum; // false (strict equality, different types)
let isNotEqualLoose = num1 != strNum; // false
let isNotEqualStrict = num1 !== strNum; // true
let isGreater = num1 > num2; // true
let isLess = num1 < num2; // false
let isGreaterOrEqual = num1 >= 10; // true
let isLessOrEqual = num2 <= 5; // true

Avoid Loose Equality

Always use === and !== (strict equality) instead of == and != (loose equality) to avoid unexpected type coercion bugs. For example, 0 == false is true, but 0 === false is false.

Logical Operators

Logical operators are used to determine the logic between variables or values. They are often used with boolean values and return a boolean result.

Logical AND &&

Returns true if both operands are true

Logical OR ||

Returns true if at least one operand is true

Logical NOT !

Returns the opposite boolean value (inverts)

Nullish Coalescing ??

Returns right operand if left is null/undefined

// Logical operator examples
let isLogged = true, hasPermission = false, age = 25;

// Logical AND (both conditions must be true)
let canAccess = isLogged && age >= 18; // true

// Logical OR (at least one condition must be true)
let canProceed = isLogged || hasPermission; // true

// Logical NOT (inverts the boolean value)
let isNotLogged = !isLogged; // false

// Nullish coalescing (ES2020)
let username = null ?? "Guest"; // "Guest"
let score = 0 ?? 10; // 0 (0 is not null or undefined)

Short-Circuit Evaluation

Logical operators use short-circuit evaluation. For &&, if the first operand is false, the second operand won't be evaluated. For ||, if the first operand is true, the second won't be evaluated.

Conditional (Ternary) Operator

The conditional (ternary) operator is the only JavaScript operator that takes three operands. It's a shorthand for an if-else statement.

// Syntax: condition ? exprIfTrue : exprIfFalse

// Traditional if-else statement
let age = 18;
let message;

if (age >= 18) {
  message = "You are an adult";
} else {
  message = "You are a minor";
}

// Same logic with ternary operator
let ternaryMessage = age >= 18 ? "You are an adult" : "You are a minor";

When to Use the Ternary Operator

Use the ternary operator for simple conditional assignments. For more complex conditions with multiple branches, stick with if-else statements for better readability.

// More ternary operator examples
let score = 85;
let grade = score >= 90 ? "A" :
                 score >= 80 ? "B" :
                 score >= 70 ? "C" :
                 score >= 60 ? "D" : "F";
// grade is "B"

// Ternary operator for function call
let isMember = true;
let discount = isMember ? calculateMemberDiscount() : calculateRegularDiscount();

Operator Precedence

Operator precedence determines the order in which operators are evaluated when an expression has multiple operators. Operators with higher precedence are evaluated first.

// Operator precedence examples
let result1 = 5 + 3 * 2; // 11 (multiplication before addition)
let result2 = (5 + 3) * 2; // 16 (parentheses change precedence)
let result3 = 2 ** 3 * 2; // 16 (exponentiation before multiplication)
let result4 = 10 > 5 && 3 < 4; // true (comparisons before logical AND)
Precedence Operator Type Associativity Example Operators
Highest Grouping n/a ()
High Member Access left-to-right . []
High Function Call left-to-right ()
High Increment/Decrement n/a ++ --
High Logical NOT right-to-left !
Medium Multiplication/Division left-to-right * / %
Medium Addition/Subtraction left-to-right + -
Low Comparison left-to-right < > <= >=
Low Equality left-to-right == != === !==
Low Logical AND left-to-right &&
Low Logical OR left-to-right ||
Lowest Assignment right-to-left = += -= etc.

Best Practice

When in doubt about operator precedence, use parentheses () to explicitly define the order of operations. This makes your code clearer and prevents bugs.

Summary & Key Takeaways

What You've Learned

  • Operators are symbols that perform operations on values (operands)
  • Expressions are combinations of values, variables, and operators that evaluate to a single value
  • JavaScript has arithmetic, assignment, comparison, logical, and conditional operators
  • Operator precedence determines the order of evaluation in complex expressions
  • Always use strict equality (=== and !==) to avoid type coercion bugs
  • The ternary operator provides a concise way to write simple conditional statements

Understanding operators and expressions is fundamental to JavaScript programming. With this knowledge, you can now write more complex and powerful expressions to manipulate data and control program flow.
