Introduction to JavaScript Events

In JavaScript, events are actions or occurrences that happen in the browser, which the browser tells you about so you can respond to them in some way. Events are the cornerstone of interactive web applications.

// Example: Responding to a button click event
const button = document.querySelector('button');

button.addEventListener('click', function() {
  // This code runs when the button is clicked
  console.log('Button was clicked!');
});

When an event occurs, the browser creates an event object containing details about the event, and calls any event handlers (functions) that are listening for that event.

Event Handlers & Listeners

There are several ways to handle events in JavaScript. The modern approach is to use addEventListener().

Three Ways to Handle Events

// 1. HTML attribute (not recommended for complex applications)
<button onclick="handleClick()">Click me</button>

// 2. DOM property (can only assign one handler)
button.onclick = function() {
  console.log('Button clicked!');
};

// 3. addEventListener() (recommended - can add multiple handlers)
button.addEventListener('click', handleClick);
button.addEventListener('click', anotherHandler); // Multiple handlers possible

Best Practice

Always use addEventListener() instead of HTML attributes or DOM properties because:

  • You can add multiple event handlers
  • You have more control (can use capturing phase)
  • It's easier to remove event listeners when needed
  • It keeps your JavaScript separate from your HTML

Removing Event Listeners

// To remove an event listener, you need a named function
function handleClick() {
  console.log('Button clicked!');
}

// Add the event listener
button.addEventListener('click', handleClick);

// Later, remove the event listener
button.removeEventListener('click', handleClick);

The Event Object

When an event occurs, the browser creates an event object and passes it as an argument to the event handler. This object contains information about the event and methods to control its behavior.

// The event object is automatically passed to the handler
element.addEventListener('click', function(event) {
  // 'event' is the event object
  console.log('Event type:', event.type); // "click"
  console.log('Target element:', event.target); // The element that was clicked
  console.log('Current target:', event.currentTarget); // The element with the listener
  console.log('Mouse coordinates:', event.clientX, event.clientY);
});

Common Event Object Properties

Property Description Example Events
type The type of event (e.g., "click", "keydown") All events
target The element that triggered the event All events
currentTarget The element that the event listener is attached to All events
preventDefault() Prevents the default browser action Form submit, link clicks
stopPropagation() Stops the event from bubbling up All events
clientX / clientY Mouse coordinates relative to viewport Mouse events

shiftKey Whether Shift key was pressed Mouse, keyboard events

Mouse Events

Mouse events occur when the user interacts with a pointing device (mouse, touchpad, etc.) on an element.

click

Fires when the mouse is pressed and released on the same element

dblclick

Fires when the mouse is double-clicked on an element

mousedown

Fires when the mouse button is pressed down

mouseup

Fires when the mouse button is released

mouseover

Fires when the mouse enters an element

mouseout

Fires when the mouse leaves an element

mousemove

Fires when the mouse moves over an element

contextmenu

Fires when the right mouse button is clicked (context menu)

// Mouse event examples
const box = document.getElementById('myBox');

box.addEventListener('click', function(e) {
  console.log(`Clicked at (${e.clientX}, ${e.clientY})`);
});

box.addEventListener('mouseover', function() {
  box.style.backgroundColor = 'lightblue';
});

box.addEventListener('mouseout', function() {
  box.style.backgroundColor = '';
});

Keyboard Events

Keyboard events occur when the user presses or releases keys on the keyboard. They're essential for handling user input and creating keyboard shortcuts.

keydown

Fires when a key is pressed down

keyup

Fires when a key is released

keypress

Fires when a key that produces a character is pressed (deprecated)

Note about keypress

The keypress event is deprecated and should not be used in new code. Use keydown instead. The main difference is that keydown fires for all keys, while keypress only fired for keys that produce a character.

// Keyboard event examples
document.addEventListener('keydown', function(event) {
  console.log(`Key pressed: ${event.key}`);
  console.log(`Key code: ${event.code}`);
  console.log(`Shift pressed? ${event.shiftKey}`);

  // Example: Detect Ctrl+S for save
  if (event.ctrlKey && event.key === 's') {
    event.preventDefault(); // Prevent browser save dialog
    saveDocument(); // Custom save function
  }
});

Event Propagation

Event propagation describes how events travel through the DOM tree. There are three phases:

  1. Capturing Phase: Event travels from window down to target element
  2. Target Phase: Event reaches the target element
  3. Bubbling Phase: Event bubbles up from target to window

Event Propagation Flow

Window (Capturing)
↓
Document (Capturing)
↓
Target Element
↑
Document (Bubbling)
↑
Window (Bubbling)
// By default, events use bubbling phase (false)
outer.addEventListener('click', function() {
  console.log('Outer element clicked (bubbling)');
}, false);

// To use capturing phase, set third parameter to true
outer.addEventListener('click', function() {
  console.log('Outer element clicked (capturing)');
}, true);

// Stop event propagation
inner.addEventListener('click', function(event) {
  event.stopPropagation(); // Prevents event from bubbling up
  console.log('Inner element clicked (propagation stopped)');
});

Event Delegation

Event delegation is a technique where you attach a single event listener to a parent element to handle events for multiple child elements. This is efficient and works for dynamically added elements.

// Without event delegation (inefficient for many items)
const items = document.querySelectorAll('.item');
items.forEach(function(item) {
  item.addEventListener('click', handleClick);
});

// WITH event delegation (efficient, works for dynamic content)
const list = document.getElementById('itemList');
list.addEventListener('click', function(event) {
  // Check if the clicked element is an item
  if (event.target.matches('.item')) {
    handleClick(event);
  }
});

Benefits of Event Delegation

  • Performance: Only one event listener instead of many
  • Memory efficiency: Less memory usage
  • Dynamic content: Works for elements added after page load
  • Simpler code: Easier to manage event handling

Summary & Key Takeaways

What You've Learned

  • Events are actions that occur in the browser that JavaScript can respond to
  • Use addEventListener() to handle events (not onclick attributes)
  • The event object contains details about the event and methods to control it
  • Events propagate in three phases: capturing, target, and bubbling
  • Event delegation is an efficient pattern for handling events on multiple elements
  • Common event types include mouse, keyboard, form, and window events
