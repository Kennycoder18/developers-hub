HTML Tables Tutorial

Master HTML tables to organize and present tabular data effectively with proper semantics and accessibility.

25 min read Data Presentation Accessibility Focus

HTML Table Fundamentals

HTML tables are used to display tabular dataâ€”information that's logically arranged in rows and columns. Tables should be used for data presentation, not for page layout.

Key Concept: Tables consist of rows (<tr>), header cells (<th>), and data cells (<td>). All are contained within a <table> element.

Basic Table Structure

The simplest table consists of rows, header cells, and data cells:

<table>
  <tr>
    <th>Header 1</th>
    <th>Header 2</th>
    <th>Header 3</th>
  </tr>

  <tr>
    <td>Row 1, Cell 1</td>
    <td>Row 1, Cell 2</td>
    <td>Row 1, Cell 3</td>
  </tr>

  <tr>
    <td>Row 2, Cell 1</td>
    <td>Row 2, Cell 2</td>
    <td>Row 2, Cell 3</td>
  </tr>
</table>

Live Table Example:

Header 1 Header 2 Header 3
Row 1, Cell 1 Row 1, Cell 2 Row 1, Cell 3
Row 2, Cell 1 Row 2, Cell 2 Row 2, Cell 3

Table Usage Warning

Avoid using tables for page layout! Tables should only be used for tabular data. Use CSS Flexbox or Grid for page layouts. Table layouts break accessibility and are difficult to maintain.

Table Elements Reference

HTML provides several elements for creating structured, semantic tables:

<table>

The container element for all table content. Defines the table structure.

<table>...</table>

Use: Wraps all table elements

<tr>

Table Row. Contains table cells (<th> or <td>).

<tr>
  <td>Cell</td>
</tr>

Use: Each row of the table

<th>

Table Header. Defines a header cell. Screen readers use these for context.

<th>Column Name</th>

Use: Column or row headers

<td>

Table Data. Defines a standard data cell in the table.

<td>Data Value</td>

Use: Regular table cells with data

Structured Tables

For complex tables, HTML provides semantic elements to group table sections logically: <thead>, <tbody>, and <tfoot>.

Key Concept: These structural elements improve accessibility, allow for independent styling of table sections, and enable scrolling of the table body while keeping headers and footers fixed.

Table Sections

Well-structured tables use these grouping elements for clarity and accessibility:

<table>

  <!-- Table header -->
  <thead>
    <tr>
      <th>Product</th>
      <th>Price</th>
      <th>Quantity</th>
      <th>Total</th>
    </tr>
  </thead>

  <!-- Table body -->
  <tbody>
    <tr>
      <td>Laptop</td>
      <td>$999</td>
      <td>2</td>
      <td>$1,998</td>
    </tr>
    <tr>
      <td>Mouse</td>
      <td>$25</td>
      <td>5</td>
      <td>$125</td>
    </tr>
  </tbody>

  <!-- Table footer -->
  <tfoot>
    <tr>
      <td colspan="3">Grand Total</td>
      <td>$2,123</td>
    </tr>
  </tfoot>
</table>

Spanning Rows and Columns

The colspan and rowspan attributes allow cells to span multiple columns or rows:

Column Spanning (colspan)

A cell can span multiple columns using colspan:

<table>
  <tr>
    <th colspan="2">Combined Header</th>
  </tr>
  <tr>
    <td>Column 1</td>
    <td>Column 2</td>
  </tr>
</table>
Row Spanning (rowspan)

A cell can span multiple rows using rowspan:

<table>
  <tr>
    <td rowspan="2">Spans 2 rows</td>
    <td>Row 1, Col 2</td>
  </tr>
  <tr>
    <td>Row 2, Col 2</td>
  </tr>
</table>

Captions and Column Groups

Tables can include captions and column groups for better structure and styling:

<table>

  <!-- Table caption -->
  <caption>Monthly Sales Report - Q1 2023</caption>

  <!-- Column groups for styling -->
  <colgroup>
    <col span="2" style="background-color: #f0f9ff;">
    <col style="background-color: #fef3c7;">
  </colgroup>

  <thead>
    <tr>
      <th>Product</th>
      <th>January</th>
      <th>February</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <td>Widget A</td>
      <td>1,200</td>
      <td>1,450</td>
    </tr>
  </tbody>
</table>

Advanced Table Features

Modern HTML tables include features for accessibility, responsiveness, and complex data presentation.

Key Concept: Use scope attributes for accessibility, CSS for responsive tables, and semantic markup to ensure tables are usable by everyone, including screen reader users.

Accessible Tables

Accessibility is crucial for tables. Use scope, headers, and <caption> to make tables usable for screen readers:

<table>
  <!-- Descriptive caption -->
  <caption>Employee Quarterly Performance Ratings</caption>

  <thead>
    <tr>
      <th scope="col">Employee</th>
      <th scope="col">Q1 Rating</th>
      <th scope="col">Q2 Rating</th>
      <th scope="col">Q3 Rating</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <th scope="row">Alex Johnson</th>
      <td>Excellent</td>
      <td>Good</td>
      <td>Excellent</td>
    </tr>
  </tbody>
</table>
Screen Reader Experience

A screen reader would announce this table as: "Employee Quarterly Performance Ratings table with 4 columns and 1 row. Employee: Alex Johnson, Q1 Rating: Excellent, Q2 Rating: Good, Q3 Rating: Excellent."

Key accessibility attributes:

  • scope="col": Identifies column headers
  • scope="row": Identifies row headers
  • <caption>: Provides table description
  • <th> for headers: Semantic header cells

Responsive Tables

Tables can be challenging on mobile devices. Here are techniques for responsive tables:

Horizontal Scroll

Wrap tables in a container with horizontal scrolling:

<div class="table-container">
  <table class="responsive-table">
    <!-- Table content -->
  </table>
</div>

<!-- CSS -->
<style>
.table-container {
  overflow-x: auto;
}
.responsive-table {
  min-width: 600px;
}
</style>
Stack on Mobile

Use CSS to stack table rows on mobile devices:

<!-- CSS for stacked tables -->
<style>
@media (max-width: 600px) {
  table, thead, tbody, th, td, tr {
    display: block;
  }
  thead tr {
    position: absolute;
    top: -9999px;
    left: -9999px;
  }
  td {
    border: none;
    position: relative;
    padding-left: 50%;
  }
  td:before {
    position: absolute;
    left: 6px;
    content: attr(data-label);
    font-weight: bold;
  }
}
</style>

Practice Exercise

Challenge: Create a class schedule table with the following requirements:

  1. Use proper table structure with <thead>, <tbody>, and <tfoot>
  2. Include a descriptive <caption> for the table
  3. Add column groups with different background colors for time slots and class names
  4. Use colspan for time slots that span multiple periods
  5. Use rowspan for days that have the same schedule
  6. Implement proper scope attributes for accessibility
  7. Include a footer with schedule notes or disclaimers
  8. Make the table responsive with horizontal scrolling on small screens

Tip: Test your table with a screen reader or accessibility tool. Ensure that the table makes sense when read aloud and that all data relationships are clear.
