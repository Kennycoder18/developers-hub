HTML Links (Anchor Elements)

The <a> (anchor) element creates hyperlinks, which are the fundamental building blocks of web navigation.

Key Concept: Links connect web pages and resources. The href attribute specifies the destination URL, while the target attribute controls how the link opens.

Basic Link Syntax

Links are created using the <a> element with various attributes:

<!-- Basic link to another page -->
<a href="https://example.com">Visit Example.com</a>

<!-- Link opening in new tab -->
<a href="https://example.com" target="_blank">
  Open in New Tab
</a>

<!-- Link with title attribute for tooltip -->
<a href="https://example.com" title="Visit our homepage">
  Example with Tooltip
</a>

<!-- Email link -->
<a href="mailto:contact@example.com">Send Email</a>

<!-- Phone link -->
<a href="tel:+1234567890">Call Us</a>

Link Types and Relationships

HTML links can establish different types of relationships using the rel attribute:

External Links

Links to other websites should include rel="noopener noreferrer" for security.

<a href="https://external.com" rel="noopener noreferrer">

Security: Prevents the new page from accessing the window.opener property.

Download Links

The download attribute prompts users to save the linked resource.

<a href="/file.pdf" download="document.pdf">

Use: PDFs, documents, images, or any downloadable content.

Internal Links

Links to sections within the same page using ID anchors.

<a href="#section2">Go to Section 2</a>
<h2 id="section2">Section 2</h2>

Use: Table of contents, navigation within long pages.

Bookmark Links

Links that point to specific elements using the #id syntax.

<a href="#top">Back to Top</a>
<div id="top">Top of page</div>

Use: Skip links, back-to-top buttons, page navigation.

Accessible Links

Creating accessible links ensures all users can navigate your website effectively:

<!-- Good: Descriptive link text -->
<a href="/tutorials/html">
  Read our HTML tutorial
</a>

<!-- Bad: Non-descriptive link text -->
<a href="/tutorials/html">
  Click here
</a>

<!-- Good: Link with aria-label for screen readers -->
<a
  href="/tutorials/html"
  aria-label="Read our comprehensive HTML tutorial">
  HTML Tutorial
</a>

<!-- Skip link for keyboard navigation -->
<a href="#main-content" class="skip-link">
  Skip to main content
</a>

Common Link Mistakes

Avoid: "Click here" links (not descriptive), opening all links in new tabs (disrupts user flow), missing alt text on image links, and using JavaScript for navigation when plain HTML would work.

HTML Images

The <img> element embeds images into web pages. Proper image implementation is crucial for performance and accessibility.

Key Concept: Always include the alt attribute for accessibility. Use the src attribute for the image source and srcset for responsive images.

Basic Image Syntax

Images are added using the self-closing <img> tag with essential attributes:

<!-- Basic image with alt text -->
<img
  src="images/photo.jpg"
  alt="A beautiful mountain landscape at sunset"
>

<!-- Image with dimensions -->
<img
  src="images/logo.png"
  alt="Company Logo"
  width="200"
  height="100"
>

<!-- Responsive image with srcset -->
<img
  src="images/hero-small.jpg"
  srcset="images/hero-small.jpg 480w, images/hero-medium.jpg 768w, images/hero-large.jpg 1200w"
  sizes="(max-width: 600px) 480px, (max-width: 900px) 768px, 1200px"
  alt="Responsive hero image"
>

Screen Reader Experience

A screen reader would announce: "Image: A beautiful mountain landscape at sunset with snow-capped peaks under a colorful sky."

Why alt text matters: Users with visual impairments rely on descriptive alt text to understand image content.

Broken Image (Missing alt text would show filename)

Without alt text: Screen readers might read the filename or just say "image," providing no context to visually impaired users.

Advanced Image Techniques

Modern web development uses advanced techniques for better performance and user experience:

Figure and Figcaption

Use <figure> and <figcaption> for images with captions:

<figure>
  <img
    src="chart.png"
    alt="Bar chart showing quarterly revenue growth"
  >
  <figcaption>
    Figure 1: Quarterly revenue growth for 2023
  </figcaption>
</figure>
Responsive Images

The <picture> element provides art direction control:

<picture>
  <!-- WebP for modern browsers -->
  <source
    srcset="image.webp"
    type="image/webp"
  >
  <!-- Fallback for older browsers -->
  <source
    srcset="image.jpg"
    type="image/jpeg"
  >
  <!-- Default image -->
  <img
    src="image.jpg"
    alt="Responsive image with modern formats"
  >
</picture>

Image Attributes Reference

Understanding all available image attributes helps create optimal implementations:

Attribute Description Required
src Source URL of the image Yes
alt Alternative text for accessibility Yes*
width & height Dimensions (helps prevent layout shift) Recommended
srcset Multiple image sources for different resolutions For responsive images
sizes Image display sizes for different viewports With srcset
loading Controls lazy loading ("lazy" or "eager") For performance
decoding Hint for image decoding ("async", "sync", "auto") For performance

*Alt text exceptions: Only omit alt text if the image is purely decorative and adds no informational value. Even then, use alt="" (empty string) so screen readers skip it.

HTML Audio & Video

HTML5 introduced native <audio> and <video> elements for embedding media without plugins like Flash.

Key Concept: Provide multiple source formats for cross-browser compatibility. Always include fallback content for browsers that don't support these elements.

Audio Element

The <audio> element embeds sound content into web pages:

<!-- Basic audio player -->
<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
  Your browser does not support the audio element.
</audio>

<!-- Audio with additional attributes -->
<audio
  controls
  autoplay
  loop
  muted
  preload="metadata">
  <source src="music.mp3" type="audio/mpeg">
</audio>

Video Element

The <video> element embeds video content with built-in controls:

<!-- Basic video player -->
<video controls width="640">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Your browser does not support the video tag.
</video>

<!-- Video with poster and tracks -->
<video
  controls
  width="800"
  height="450"
  poster="thumbnail.jpg">
  <source src="movie.mp4" type="video/mp4">
  <!-- Caption track -->
  <track
    kind="captions"
    src="captions.vtt"
    srclang="en"
    label="English"
  >
  Your browser does not support HTML5 video.
</video>

Complete Media Example

Here's a complete example showing links, images, and media working together in a portfolio page:

<!-- Portfolio Section with Media -->
<section class="portfolio">
  <h2>My Photography Portfolio</h2>

  <div class="gallery">
    <figure>
      <a href="large-landscape.jpg">
        <img
          src="landscape-thumb.jpg"
          alt="Mountain landscape at golden hour"
          loading="lazy"
        >
      </a>
      <figcaption>
        <a href="#landscape-details">View Details</a>
      </figcaption>
    </figure>

    <figure>
      <img
        src="portrait.jpg"
        alt="Portrait of a woman in traditional clothing"
        srcset="portrait-480.jpg 480w, portrait-800.jpg 800w"
        sizes="(max-width: 600px) 480px, 800px"
      >
      <figcaption>Portrait Photography</figcaption>
    </figure>
  </div>

  <div class="media-section">
    <h3>Behind the Scenes Video</h3>
    <video controls poster="video-thumb.jpg">
      <source src="bts.mp4" type="video/mp4">
      <source src="bts.webm" type="video/webm">
      <track src="captions.vtt" kind="captions" srclang="en">
      Your browser doesn't support HTML5 video.
    </video>
    <p><a href="bts.mp4" download>Download Video</a></p>
  </div>

  <div class="external-links">
    <h3>Connect With Me</h3>
    <ul>
      <li>
        <a href="https://instagram.com/myportfolio"
          rel="noopener noreferrer"
          target="_blank">
          Follow on Instagram
        </a>
      </li>
      <li>
        <a href="mailto:contact@myportfolio.com">
          Email for inquiries
        </a>
      </li>
    </ul>
  </div>
</section>

Practice Exercise

Challenge: Create a blog post with rich media content:

  1. Add a featured image with proper alt text and responsive attributes
  2. Create a table of contents with internal links to article sections
  3. Embed a relevant YouTube video using an iframe (research how)
  4. Add an audio clip of an interview or podcast excerpt
  5. Include external links to references with proper rel attributes
  6. Create an image gallery with clickable thumbnails
  7. Add social sharing links (Twitter, Facebook, LinkedIn)
  8. Include a "back to top" link that uses smooth scrolling

Tip: Test all links and media elements for accessibility. Use proper contrast ratios for linked text and ensure all media has appropriate alternatives.
