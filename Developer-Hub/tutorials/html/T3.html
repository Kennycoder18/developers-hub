HTML Form Fundamentals

HTML forms are used to collect user input. The <form> element defines a form, and it contains various input elements like text fields, checkboxes, radio buttons, and more.

Key Concept: Forms send data to a server for processing. The action attribute specifies where to send the form data, and the method attribute specifies the HTTP method (GET or POST).

Basic Form Structure

Every HTML form follows this basic structure with essential attributes:

<form action="/submit-form" method="POST">

  <!-- Text input -->
  <label for="name">Name:</label>
  <input type="text" id="name" name="user_name">

  <!-- Email input -->
  <label for="email">Email:</label>
  <input type="email" id="email" name="user_email">

  <!-- Submit button -->
  <button type="submit">Submit</button>

</form>

Form Accessibility

Always: Use the <label> element with the for attribute matching the input's id. This makes forms accessible to screen readers and improves usability for everyone.

HTML Input Types

HTML5 introduced many new input types that provide better user experience and built-in validation. Each input type serves a specific purpose.

Key Concept: The type attribute of the <input> element determines what kind of data it accepts and how it's displayed. Modern browsers provide special interfaces for different input types.

Common Input Types

Here are the most commonly used input types with examples:

Text Inputs

text, password, email, tel, url, search

<input type="text" placeholder="Enter text">
<input type="password" placeholder="Password">
<input type="email" placeholder="email@example.com">

Date & Time Inputs

date, time, datetime-local, month, week

<input type="date">
<input type="time">
<input type="datetime-local">

Range & Number Inputs

number, range, color

<input type="number" min="1" max="100">
<input type="range" min="0" max="100">
<input type="color" value="#0d9488">

Selection Inputs

checkbox, radio, select, textarea

<input type="checkbox" id="option1">
<input type="radio" name="group" value="1">
<select><option>Choice</option></select>

File Inputs

file, image, hidden

<input type="file" accept=".jpg,.png">
<input type="image" src="button.png">
<input type="hidden" name="token" value="abc123">

Button Inputs

button, submit, reset

<input type="button" value="Click me">
<input type="submit" value="Send">
<input type="reset" value="Clear">

Input Attributes

Input attributes control behavior, validation, and appearance of form elements:

<form>
  <!-- Required field -->
  <input type="text" required>

  <!-- Placeholder text -->
  <input type="text" placeholder="Enter your name">

  <!-- Minimum and maximum values -->
  <input type="number" min="1" max="100">

  <!-- Pattern validation -->
  <input type="text" pattern="[A-Za-z]{3}" title="Three letter code">

  <!-- Disabled and readonly -->
  <input type="text" value="Cannot edit" readonly>
  <input type="text" value="Disabled" disabled>

  <!-- Autocomplete and autofocus -->
  <input type="email" autocomplete="email" autofocus>
</form>

Important: HTML5 form validation attributes like required, pattern, min, max, and type="email" provide client-side validation without JavaScript.

Advanced Form Features

Modern HTML forms include advanced features for better user experience, accessibility, and data handling.

Key Concept: HTML5 introduced the <datalist> element for autocomplete, <output> for calculated results, and <progress>/<meter> for displaying values.

Form Structure Elements

HTML5 introduced semantic elements for better form organization:

Fieldset & Legend

Group related form elements together:

<fieldset>
  <legend>Contact Information</legend>

  <label for="phone">Phone:</label>
  <input type="tel" id="phone" name="phone">

  <label for="email">Email:</label>
  <input type="email" id="email" name="email">
</fieldset>
Datalist for Autocomplete

Provide suggested options for an input field:

<label for="browser">Choose your browser:</label>
<input list="browsers" name="browser" id="browser">

<datalist id="browsers">
  <option value="Chrome">
  <option value="Firefox">
  <option value="Safari">
  <option value="Edge">
  <option value="Opera">
</datalist>

Form Validation

HTML5 provides built-in form validation that works without JavaScript:

<form id="validatedForm">
  <label for="username">Username (required, 3-20 chars):</label>
  <input type="text" id="username" name="username"
         required minlength="3" maxlength="20">

  <label for="age">Age (18-100):</label>
  <input type="number" id="age" name="age" min="18" max="100">

  <label for="website">Website URL:</label>
  <input type="url" id="website" name="website">

  <label for="zip">ZIP Code (5 digits):</label>
  <input type="text" id="zip" name="zip"
         pattern="[0-9]{5}" title="Five digit ZIP code">

  <button type="submit">Validate & Submit</button>
</form>

Client-Side vs Server-Side Validation

Remember: HTML5 validation is client-side only. You must always validate form data on the server as well, as client-side validation can be bypassed.

Complete Form Example

Here's a complete, functional contact form with all the concepts we've covered:

<!-- Complete Contact Form -->
<form action="/contact" method="POST" id="contactForm">

  <fieldset>
    <legend>Personal Information</legend>

    <div class="form-row">
      <div class="form-group">
        <label for="firstName">First Name *</label>
        <input type="text" id="firstName" name="firstName"
             required autofocus>
      </div>

      <div class="form-group">
        <label for="lastName">Last Name *</label>
        <input type="text" id="lastName" name="lastName" required>
      </div>
    </div>

    <div class="form-group">
      <label for="email">Email Address *</label>
      <input type="email" id="email" name="email"
            required placeholder="your.email@example.com">
    </div>
  </fieldset>

  <fieldset>
    <legend>Message Details</legend>

    <div class="form-group">
      <label for="subject">Subject</label>
      <input list="subjects" id="subject" name="subject">
      <datalist id="subjects">
        <option value="General Inquiry">
        <option value="Technical Support">
        <option value="Feedback">
        <option value="Partnership">
      </datalist>
    </div>

    <div class="form-group">
      <label for="message">Message *</label>
      <textarea id="message" name="message" rows="5"
            required minlength="10"></textarea>
    </div>

    <div class="form-group">
      <label>Priority:</label>
      <select name="priority">
        <option value="low">Low</option>
        <option value="normal" selected>Normal</option>
        <option value="high">High</option>
      </select>
    </div>
  </fieldset>

  <div class="form-group">
    <button type="submit" class="btn-submit">Send Message</button>
    <button type="reset" class="btn-reset">Clear Form</button>
  </div>
</form>
