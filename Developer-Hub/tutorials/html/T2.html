HTML Attributes & Headings

Master the power of HTML attributes and proper heading hierarchy to create accessible, well-structured web pages.

25 min read Core HTML Concepts Accessibility Focus

Understanding HTML Headings

HTML headings are defined with the <h1> to <h6> tags. They create a visual hierarchy and semantic structure for your content, which is essential for both users and search engines.

Key Concept: Headings provide a hierarchical structure to your content. <h1> represents the main heading of the page (usually the page title), while <h2> to <h6> represent subheadings of decreasing importance.

Heading Hierarchy & Best Practices

Proper heading hierarchy is crucial for accessibility and SEO. Here's how headings should be structured:

<h1>Main Page Title (Only one per page)</h1>

<h2>Primary Section Title</h2>
<h3>Subsection Title</h3>
<!-- Content for this subsection -->
<p>Paragraph text...</p>

<h3>Another Subsection Title</h3>
<p>More content...</p>

<h2>Another Primary Section</h2>
<h3>Subsection</h3>
<h4>Nested Subsection (if needed)</h4>

Common Heading Mistakes

Avoid: Skipping heading levels (e.g., going from <h2> directly to <h4>). Do: Maintain a logical hierarchy. Avoid: Using headings for styling alone. Do: Use CSS for visual appearance.

Heading Visual Hierarchy

Note: While browsers apply default styles to headings, you should always use CSS to control their appearance. The semantic meaning (h1-h6) is more important than the visual style.

What are HTML Attributes?

HTML attributes provide additional information about HTML elements. They are always specified in the opening tag and usually come in name/value pairs like: name="value".

Key Concept: Attributes modify HTML elements or provide additional functionality. Common attributes include id, class, src, href, alt, and style.

Common HTML Attributes

Here are some of the most frequently used HTML attributes across different elements:

id & class

The id attribute specifies a unique identifier, while class specifies one or more class names for an element.

<div id="header" class="container primary">

Use: CSS styling, JavaScript manipulation

src & alt

The src attribute specifies the source of an image, while alt provides alternative text for accessibility.

<img src="logo.png" alt="Company Logo">

Use: Images, media elements

href & target

The href attribute specifies the URL for links, while target defines where to open the linked document.

<a href="https://example.com" target="_blank">

Use: Anchor tags (links)

style

The style attribute applies inline CSS styles to an element.

<p style="color: blue; font-size: 16px;">

Note: It's better to use external CSS files for styling.

title

The title attribute provides extra information about an element (shown as a tooltip).

<abbr title="HyperText Markup Language">HTML</abbr>

Use: Tooltips, accessibility

placeholder & value

The placeholder attribute shows hint text in form inputs, while value specifies the initial value.

<input type="text" placeholder="Enter name" value="John">

Use: Form elements

Global Attributes

Global attributes can be used on any HTML element. Here are some important ones:

<!-- Global attributes example -->
<div
  id="uniqueId"
  class="container highlighted"
  title="Tooltip information"
  data-custom="custom data"
  style="color: blue;"
  hidden
  contenteditable="true"
>
  Content goes here
</div>

Important: The data-* attributes allow you to store custom data private to the page or application. These are extremely useful for JavaScript functionality.

Headings with Attributes

Attributes enhance headings by adding functionality, accessibility features, and styling hooks. Here's how to combine them effectively:

Accessible Headings with IDs

Adding IDs to headings creates anchor links for easy navigation:

<h2 id="introduction">Introduction</h2>
<p>Content for introduction section...</p>

<h2 id="methods">Methods</h2>
<p>Content for methods section...</p>

<!-- Navigation links to sections -->
<a href="#introduction">Go to Introduction</a>
<a href="#methods">Go to Methods</a>
Styled Headings with Classes

Use class attributes to apply consistent styling to headings:

<h1 class="page-title primary-color">Welcome</h1>
<h2 class="section-title">About Us</h2>
<h3 class="subsection-title with-border">Our Mission</h3>

<!-- Corresponding CSS -->
<style>
.page-title { font-size: 2.5rem; }
.section-title { color: #0d9488; }
.with-border { border-bottom: 2px solid #0ea5e9; }
</style>
Headings with Data Attributes

Data attributes can store extra information for JavaScript interaction:

<h2
  data-section="user-profile"
  data-collapsible="true"
  data-priority="high"
>
  User Profile Settings
</h2>

<!-- JavaScript can access these values -->
<script>
const heading = document.querySelector('h2');
console.log(heading.dataset.section); // "user-profile"
console.log(heading.dataset.collapsible); // "true"
</script>

Complete Example: Blog Post

Here's a complete example showing headings and attributes working together in a blog post:

<article class="blog-post" id="post-42">
  <header>
    <h1 class="post-title" data-post-category="tutorial">
      Mastering HTML Attributes & Headings
    </h1>
    <p class="post-meta">
      Posted on <time datetime="2023-10-15">October 15, 2023</time>
    </p>
  </header>

  <div class="post-content">
    <h2 id="intro-heading">Introduction</h2>
    <p>HTML attributes and headings work together...</p>

    <h2 id="headings-section">HTML Headings Explained</h2>
    <h3>Heading Hierarchy</h3>
    <p>Proper hierarchy is essential...</p>

    <h3>Accessibility Considerations</h3>
    <p>Screen readers use heading structure...</p>

    <h2 id="attributes-section">Common HTML Attributes</h2>
    <p>Attributes add functionality...</p>
  </div>

  <footer class="post-footer">
    <div class="post-tags">
      <a href="/tag/html" class="tag">HTML</a>
      <a href="/tag/webdev" class="tag">Web Development</a>
    </div>
  </footer>
</article>

Practice Exercise

Challenge: Create a product page with proper headings and attributes:

  1. Use <h1> for the product name with an appropriate class
  2. Create sections with <h2> headings that have unique IDs
  3. Add <h3> headings for subsections within each main section
  4. Include data attributes to store product information (price, category, rating)
  5. Add a product image with proper src and alt attributes

Tip: Remember that heading hierarchy should be logical, and attributes should provide useful information or functionality.
