What is the DOM?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a document as a tree of objects, where each node is an object representing a part of the document.

DOM Tree Structure

Document (Root)
HTML
HEAD
BODY
DIV
P
UL
// The DOM represents this HTML as a tree of objects:
<!DOCTYPE html>
<html>
  <head>
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello DOM!</h1>
    <p>This is a paragraph.</p>
  </body>
</html>

Important Distinction

The DOM is NOT the same as HTML source code. The DOM is a live representation that can be modified by JavaScript, while HTML is the static source code. When you modify the DOM, the browser updates what the user sees without changing the original HTML file.

Selecting DOM Elements

Before you can manipulate elements, you need to select them. JavaScript provides several methods to select elements from the DOM.

getElementById()

Selects an element by its ID attribute

querySelector()

Selects the first element matching a CSS selector

querySelectorAll()

Selects all elements matching a CSS selector

getElementsByClassName()

Selects elements by class name

getElementsByTagName()

Selects elements by tag name

// Different ways to select elements
// HTML: <div id="main" class="container">...</div>
// <p class="text">Paragraph 1</p>
// <p class="text">Paragraph 2</p>

const byId = document.getElementById('main');
const byQuery = document.querySelector('.container');
const allByQuery = document.querySelectorAll('.text');
const byClass = document.getElementsByClassName('text');
const byTag = document.getElementsByTagName('p');

Important Differences

querySelectorAll() returns a NodeList (static snapshot), while getElementsByClassName() and getElementsByTagName() return HTMLCollections (live collections). NodeLists have forEach method, HTMLCollections don't.

Modifying Element Content

Once you've selected elements, you can modify their content using various properties and methods.

innerHTML

Gets or sets HTML content inside an element

textContent

Gets or sets text content (safer than innerHTML)

innerText

Gets or sets visible text content

value

Gets or sets value of form elements

// Modifying element content
const element = document.getElementById('myElement');

// Change HTML content
element.innerHTML = '<strong>Bold Text</strong> and <em>italic</em>';

// Change text content (safer - no HTML parsing)
element.textContent = 'Plain text without HTML';

// For form elements
const input = document.getElementById('myInput');
input.value = 'New input value';

Security Best Practice

Use textContent instead of innerHTML when you're only dealing with text content. innerHTML can expose your site to Cross-Site Scripting (XSS) attacks if you insert user-provided content without proper sanitization.

Creating and Removing Elements

You can dynamically create new elements and add them to the DOM, or remove existing elements.

createElement()

Creates a new element node

appendChild()

Appends a node as the last child

insertBefore()

Inserts a node before a reference node

removeChild()

Removes a child node

remove()

Removes the element itself

// Creating and adding elements
// 1. Create a new element
const newDiv = document.createElement('div');
newDiv.textContent = 'I am a new div!';
newDiv.className = 'my-class';

// 2. Get the parent element
const container = document.getElementById('container');

// 3. Append the new element
container.appendChild(newDiv);

// 4. Insert before a specific element
const referenceElement = document.getElementById('reference');
container.insertBefore(newDiv, referenceElement);

// 5. Remove an element
container.removeChild(newDiv);
// OR using the newer remove() method
newDiv.remove();

Working with Attributes & Classes

DOM elements have attributes (id, src, href, etc.) and classes that can be manipulated programmatically.

getAttribute()

Gets the value of an attribute

setAttribute()

Sets the value of an attribute

classList.add()

Adds one or more classes

classList.remove()

Removes one or more classes

classList.toggle()

Toggles a class (adds if missing, removes if present)

// Working with attributes and classes
const image = document.getElementById('myImage');

// Get an attribute value
const src = image.getAttribute('src');

// Set an attribute value
image.setAttribute('src', 'new-image.jpg');
image.setAttribute('alt', 'New image description');

// Working with classes (modern approach)
const box = document.getElementById('myBox');

// Add classes
box.classList.add('highlight', 'active');

// Remove classes
box.classList.remove('highlight');

// Toggle a class
box.classList.toggle('active'); // If active exists, removes it; otherwise adds it

// Check if a class exists
const isActive = box.classList.contains('active');

Styling Elements Dynamically

You can change the style of elements directly using the style property, or by adding/removing CSS classes.

// Changing element styles
const element = document.getElementById('myElement');

// Direct style manipulation (camelCase property names)
element.style.backgroundColor = 'blue';
element.style.color = 'white';
element.style.padding = '20px';
element.style.fontSize = '18px';
element.style.border = '2px solid black';

// Using CSS classes (better approach for complex styles)
element.classList.add('styled-element');

// Getting computed styles
const computedStyle = window.getComputedStyle(element);
const actualColor = computedStyle.getPropertyValue('color');

Best Practice for Styling

Use CSS classes (classList.add/remove/toggle) instead of direct style manipulation when possible. This separates concerns (CSS for styling, JavaScript for behavior) and makes your code more maintainable. Use direct style manipulation only for dynamic values that can't be predefined in CSS.

Summary & Key Takeaways

What You've Learned

  • The DOM is a tree representation of an HTML document
  • Use querySelector and querySelectorAll for flexible element selection
  • textContent is safer than innerHTML for text manipulation
  • Create elements with createElement() and add them with appendChild()
  • Use classList methods to manipulate CSS classes
  • Direct style manipulation is possible but CSS classes are often better
  • Event handling makes DOM manipulation interactive

Performance Tips

  • Batch DOM operations - Minimize reflows by making multiple changes at once
  • Use DocumentFragment for adding multiple elements efficiently
  • Cache DOM references - Store selected elements in variables
  • Debounce rapid updates - For events like scroll or resize
  • Avoid inline styles when CSS classes can achieve the same result
