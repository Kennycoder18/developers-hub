CSS Animations Introduction

CSS animations allow you to create smooth, performant animations without JavaScript. They can transform, move, fade, and rotate elements to create engaging user experiences.

Why CSS Animations Matter:
  • Performance: GPU accelerated, smoother than JavaScript animations
  • Simplicity: Declarative syntax, easier to write and maintain
  • Responsive: Work seamlessly across all devices
  • Accessibility: Can respect user motion preferences

Transitions

Simple animations between two states (like hover). Control property, duration, timing, and delay.

Keyframes

Complex animations with multiple steps. Define the entire animation sequence with @keyframes.

Properties

Control duration, timing, delay, iteration count, direction, fill mode, and play state.

Transition Properties

Transition Properties Syntax 
/* Shorthand - Most common */
transition: all 0.3s ease 0.1s;
/* property | duration | timing-function | delay */

/* Longhand - Individual properties */
transition-property: all;
transition-duration: 0.3s;
transition-timing-function: ease;
transition-delay: 0.1s;

/* Multiple properties with different timings */
transition: 
  transform 0.3s ease,
  background 0.5s linear 0.1s,
  opacity 0.2s ease-out;

Keyframes Syntax Examples 
/* Basic keyframes with from/to */
@keyframes fadeIn {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}

/* Multiple keyframes with percentages */
@keyframes bounce {
  0%, 100% {
    transform: translateY(0);
  }
  50% {
    transform: translateY(-30px);
  }
}

/* Complex animation with multiple properties */
@keyframes slideAndFade {
  0% {
    transform: translateX(-100px);
    opacity: 0;
  }
  50% {
    opacity: 0.5;
  }
  100% {
    transform: translateX(0);
    opacity: 1;
  }
}

/* Applying the animation */
.animated-element {
  animation: fadeIn 2s ease;
  /* name | duration | timing-function */
}

/* Complete animation shorthand */
.complex-animation {
  animation: bounce 1s ease-in-out 0.5s 3 alternate both running;
  /* name | duration | timing-function | delay | iteration-count | direction | fill-mode | play-state */
}

Animation Properties

Animation Properties 
/* Individual animation properties */
animation-name: slideIn;
animation-duration: 2s;
animation-timing-function: ease-in-out;
animation-delay: 0.5s;
animation-iteration-count: 3; /* or 'infinite' */
animation-direction: alternate; /* normal, reverse, alternate, alternate-reverse */
animation-fill-mode: both; /* none, forwards, backwards, both */
animation-play-state: running; /* running or paused */

/* Shorthand (recommended) */
animation: slideIn 2s ease-in-out 0.5s 3 alternate both running;

/* Multiple animations */
animation: 
  fadeIn 1s ease,
  slideIn 2s ease 1s;

Animation Best Practices:
  • Use transform and opacity for performance (GPU accelerated)
  • Keep animations under 300ms for interactions, 500ms-1000ms for entrances
  • Use will-change sparingly for elements you know will animate
  • Respect user preferences with @media (prefers-reduced-motion)
  • Test animations on different devices and browsers

Performance & Accessibility

Well-implemented animations should be both performant and accessible to all users.

Performance & Accessibility Tips 
/* GPU accelerated properties (best performance) */
.performant {
  transform: translate3d(0, 0, 0); /* Forces GPU acceleration */
  will-change: transform; /* Hint browser about upcoming changes */
}

/* Respect user motion preferences */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}

/* Optimize animations for mobile */
.mobile-optimized {
  /* Use shorter durations on mobile */
  transition-duration: 0.2s;
}

@media (min-width: 768px) {
  .mobile-optimized {
    transition-duration: 0.3s;
  }
}

/* Pause animations when not in viewport */
.pause-out-of-view {
  animation-play-state: paused;
}

.pause-out-of-view.in-view {
  animation-play-state: running;
}

/* Hardware acceleration for complex animations */
.complex-animation {
  transform: translateZ(0);
  backface-visibility: hidden;
  perspective: 1000px;
}

Key Takeaways

You've mastered CSS Animations! Here's what you should remember:
  • Use transitions for simple state changes (hover, focus)
  • Use @keyframes for complex, multi-step animations
  • Animate transform and opacity for best performance
  • Respect user preferences with prefers-reduced-motion
  • Keep animation durations appropriate (300ms for interactions)
  • Use the animation shorthand for cleaner code
  • Test animations across different devices and browsers

Continue your learning journey:

  • Experiment with different timing functions (ease, linear, cubic-bezier)
  • Learn about CSS transforms (translate, rotate, scale, skew)
  • Explore JavaScript animation libraries like GSAP for complex sequences
  • Study the principles of motion design (anticipation, follow-through)
  • Practice creating micro-interactions for better UX
